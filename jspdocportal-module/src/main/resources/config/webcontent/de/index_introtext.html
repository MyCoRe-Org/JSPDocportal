Dieser Server ist eine Beispielanwendung eines Dokumentenservers, wie er in wissenschaftlichen Bibliotheken genutzt werden kann.

Alle Dokumente werden mit einem erweiterten Dublin Core Metadatensatz in das System eingestellt und sind sowohl über diese Metadaten als auch über den Volltext recherchierbar.
Dokumente können beliebigen Klassifikationen als Ordnungskriterien zugeordnet werden. Die Klassifikationsauswahl ermöglicht Ihnen eine hierarchische Auflistung der Dokumente (Browsen).
In dieser Beispielanwendung wurde der Fokus auf die Modellierung und Integration bibliothekarischer Geschäftsprozesse gelegt.
Neben Autoren und Institutionen gibt es 2 Grundtypen von Dokumenten - Dissertationen und Publikationen.
Für Dissertationen gelten strengere Vorschriften, da sie ein Teil des Promotionsprozesses sind und nach festgelegten Regeln der jeweiligen Einrichtung vorzuhalten sind.
Für das Einstellen von Publikationen sind weniger Regeln zu beachten. Neben reinen Texten (z. B. wissenschaftliche Artikel) können auch Präsentationen, Audio- und Videodateien auf dem Server abgelegt werden.
Autoren- und Institutionsdaten können mit den zugehörigen Dokumentendaten verknüpft werden.

Die Dokumente werden archiviert, ihre Verfügbarkeit langfristig gesichert.
